Create a reservation system MVP using Next.js 14 (App Router) + Tailwind CSS + Supabase.
Business: 美容サロン「ビューティーサロン HANA」
All UI text in Japanese. Follow Japanese business culture and UX patterns.

=== JAPANESE BUSINESS CULTURE REQUIREMENTS ===
- Formal but warm language (です/ます調)
- Show trust signals: 「個人情報保護」「SSL暗号化通信」 badges in footer
- Confirmation at every step (Japanese users expect多めの確認)
- Always show 「戻る」 button (users fear making mistakes)
- Phone number prominently displayed (older users prefer calling)
- Include 「営業時間」 and 「定休日」 clearly
- Polite error messages: 「恐れ入りますが、〜をご確認ください」
- Success message with お辞儀 feeling: 「ご予約ありがとうございます」
- Use Japanese date format: 2024年1月15日（月）
- Time in 24h or with 午前/午後

=== CUSTOMER BOOKING FLOW (src/app/page.tsx) ===
Step-by-step flow with progress indicator:

Step 1: メニュー選択
- Display services from config as cards
- Show duration and price
- Allow multiple service selection
- Running total at bottom

Step 2: 日時選択
- Calendar component (current month + next month)
- Gray out past dates and closed days
- After date selection, show available time slots
- Slots already booked should show 「×」

Step 3: お客様情報入力
- Form fields:
  - お名前（必須）
  - フリガナ（必須）- Important in Japan!
  - 電話番号（必須）
  - メールアドレス（必須）
  - ご要望・備考（任意）
- Validation with polite error messages
- Show 「個人情報の取り扱いについて」 link

Step 4: 確認画面
- Show ALL details before final submission
- Editable (「変更する」 buttons for each section)
- Cancellation policy reminder
- Final 「予約を確定する」 button (make it prominent but not scary)

Step 5: 完了画面
- 「ご予約ありがとうございます」
- Reservation number (e.g., HANA-20240115-001)
- Summary of booking
- 「カレンダーに追加」 button (generate .ics)
- 「ホームに戻る」 button

3. カレンダー表示 - Monthly view with booking counts

Status badges: 確定（blue）, 完了（green）, キャンセル（gray）

=== DATABASE SCHEMA (supabase/schema.sql) ===
-- Reservations table
create table reservations (
  id uuid default gen_random_uuid() primary key,
  reservation_number text unique not null,
  customer_name text not null,
  customer_name_kana text not null,
  phone text not null,
  email text not null,
  date date not null,
  time text not null,
  services jsonb not null, -- [{id, name, duration, price}]
  total_duration int not null,
  total_price int not null,
  notes text,
  status text default '確定', -- 確定, 完了, キャンセル
  created_at timestamp with time zone default now()
);

-- Index for date queries
create index idx_reservations_date on reservations(date);
